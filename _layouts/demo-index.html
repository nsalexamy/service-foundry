Automated Deployment of an Application Observability Platform

This demonstration highlights the automated deployment of a full observability platform on Kubernetes—powered by a single configuration file.

🌐 What’s Included in the Demo

This end-to-end deployment showcases how various open-source tools work together to create a robust, production-ready observability platform.

  • ✅ The runtime environment is provisioned using Amazon EKS and ECR
  • ✅ Requires the following AWS credentials: AWS_ACCESS_KEY_ID, AWS_ACCOUNT_ID, AWS_SECRET_ACCESS_KEY, and AWS_REGION
  • ✅ Deployment is managed using the Service Foundry Builder Helm chart
  • ✅ Modules include infra-foundry, o11y-foundry, and sso-foundry, working in unison
  • ✅ Keycloak is deployed and configured with a default realm using Terraform
  • ✅ Traefik is set up as the middleware for Single Sign-On (SSO)
  • ✅ cert-manager is installed to manage TLS certificates, including for OpenSearch
  • ✅ Both the Prometheus Operator and OpenTelemetry Operator are deployed
  • ✅ OpenTelemetry serves as the core of the observability platform
  • ✅ OpenTelemetry Target Allocator collects metrics from instrumented applications
  • ✅ Jaeger v2 is deployed with Apache Cassandra as its storage backend
  • ✅ OpenSearch is deployed and stores log data
  • ✅ Data Prepper is integrated with OpenTelemetry to collect and process logs
  • ✅ OpenSearch Dashboards provides a web interface for querying log data
  • ✅ Prometheus is deployed for metrics scraping
  • ✅ Grafana is set up and connected to Prometheus for visualization
  • ✅ A sample Spring Boot application (otel-spring-example) is deployed
  • ✅ OAuth2 Proxy is used for authentication
  • ✅ Traefik Ingress controllers are configured for Jaeger, Prometheus, Grafana, and OpenSearch
