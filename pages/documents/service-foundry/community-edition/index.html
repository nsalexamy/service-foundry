What is Service Foundry?

Service Foundry is an Internal Developer Platform (IDP) that enables teams to provision, deploy, secure, and observe workloads on Kubernetes in just 10 minutes.

Service Foundry is a Curated Stack of enterprise-grade open-source tools combined with a UI-driven GitOps console. As a self-service platform engineering solution, Service Foundry eliminates the complexity of assembling and integrating multiple tools by providing a pre-configured, production-ready GitOps platform.

No proprietary subscriptions or additional tools are required—everything runs on your own Kubernetes cluster using battle-tested open-source components.

Service Foundry integrates popular open-source tools into a cohesive automation system, enabling you to:

  • Use standardized GitOps patterns to manage infrastructure and application workloads declaratively

  • Collaborate effectively through version-controlled configuration stored in Git

  • Deploy complete application stacks including observability, identity management, and data pipelines

  • Secure resources using enterprise-grade Single Sign-On (SSO) and encrypted secrets management

  • Accelerate deployment with a curated Application Catalog of ready-to-use open-source applications

It bridges the gap between developers and operations by providing an intuitive UI-driven console while preserving the auditable, declarative nature of Git-based infrastructure as code.

Internal Developer Platform (IDP)

Service Foundry is an Internal Developer Platform (IDP)—a curated set of tools, services, and automated processes that enable developers to manage the entire application lifecycle independently, without needing deep expertise in Kubernetes, cloud infrastructure, or security internals.

What Problem Does an IDP Solve?

Without an IDP, developers face several bottlenecks:

  • TicketOps: Waiting days for infrastructure teams to provision databases, open firewall ports, or grant permissions

  • Complexity Overload: Needing to master Kubernetes, cloud APIs, networking, and security just to deploy a simple service

  • Inconsistent Practices: Every team builds infrastructure differently, leading to maintenance nightmares

  • Manual Deployments: Error-prone manual steps that cause "works on my machine" syndrome

An IDP removes these bottlenecks by providing self-service capabilities, standardized workflows, and automated operations.

The Five Planes of an IDP

Modern IDPs are structured into five technical "planes" that handle different aspects of the platform:

  1. Developer Control Plane: The primary interface (web console, CLI, or API) where developers interact with the platform to create services, deploy applications, and manage environments.

  2. Integration and Delivery Plane: Automates the build, test, and deployment pipeline using CI/CD tools like ArgoCD, GitHub Actions, and Argo Rollouts to implement GitOps workflows.

  3. Resource Plane: The underlying infrastructure (Kubernetes clusters, databases, storage, networking) abstracted away from developers through tools like Terraform and Crossplane.

  4. Security Plane: Manages identity, authentication, authorization (RBAC), and secrets using tools like Keycloak, OAuth2 Proxy, and Sealed Secrets.

  5. Observability Plane: Monitors platform and application health through integrated logging, metrics, and tracing using OpenTelemetry, Prometheus, Grafana, and Loki.

Service Foundry implements all five planes out of the box, providing a complete, production-ready IDP experience.

Service Foundry Key Features

Service Foundry provides enterprise-grade capabilities out of the box:

  • Auditable Infrastructure as Code: All configuration changes are tracked in Git, providing full audit trails and rollback capabilities

  • Standardized GitOps Patterns: Implement industry best practices for continuous deployment and infrastructure management

  • Version-Controlled Configuration: Manage all platform and application configurations through Git workflows

  • Complete Application Stacks: Deploy entire technology stacks (observability, identity, data pipelines) with pre-configured integrations

  • Enterprise Security: Built-in Single Sign-On (SSO) and encrypted secrets management using Keycloak and Sealed Secrets

  • Automated Certificate Management: Automatic TLS certificate provisioning and renewal via cert-manager

Main Components

Service Foundry is built on a curated stack of proven open-source tools, carefully integrated to work seamlessly together. These components map directly to the five IDP planes:

Core Components

Developer Control Plane

  • Service Foundry Console: UI-driven GitOps console providing visual management of applications, deployments, and platform configurations

Integration and Delivery Plane

  • ArgoCD: GitOps continuous delivery tool for Kubernetes, providing declarative application deployment and lifecycle management

  • Argo Rollouts: Advanced deployment strategies including blue-green, canary, and progressive delivery with automated rollbacks

  • Service Foundry Builder: GitOps resource generator that creates and updates Kubernetes manifests and Helm charts

Resource Plane

  • Traefik: Modern cloud-native ingress controller with Gateway API support for routing external traffic to services

  • PostgreSQL: Relational database for platform data storage and application backends

  • Redis: In-memory data store for user session management and application caching

Security Plane

  • Keycloak: Enterprise-grade Identity and Access Management (IAM) for centralized authentication and authorization

  • OAuth2 Proxy: Secure Single Sign-On (SSO) integration layer that protects applications with authentication

  • Sealed Secrets: Encrypted secrets management allowing safe storage of sensitive data in Git repositories

  • Cert-manager: Automated TLS certificate provisioning and renewal from Let’s Encrypt and other certificate authorities

Observability Plane

The observability stack is optional but highly recommended for production environments. All observability components can be easily installed through the Service Foundry Console UI.

  • OpenTelemetry: Vendor-neutral observability framework for collecting traces, metrics, and logs

  • Prometheus: Industry-standard metrics collection, storage, and alerting system

  • Grafana: Feature-rich observability dashboard for visualizing metrics, logs, and traces

  • Jaeger: Distributed tracing system for monitoring and troubleshooting microservices

  • OpenSearch: Scalable search and analytics engine for storing and querying observability data

  • Fluent Bit: Lightweight and efficient log collection and forwarding agent

  • Loki: Horizontally scalable log aggregation system designed for cloud-native environments

  • Tempo: Cost-effective distributed tracing backend with seamless Grafana integration

What You Need to Run Service Foundry

Service Foundry has minimal requirements and can be deployed in just 10 minutes with the following prerequisites:

Infrastructure Requirements

  • A Kubernetes Cluster: AWS Elastic Kubernetes Service (EKS)

    • Supported Platform: This version of Service Foundry has been tested and is currently supported only on Amazon Web Services (AWS) using EKS

    • Kubernetes Version: v1.33 or higher

    • Future Support: Additional cloud providers (GKE, AKS) and self-managed Kubernetes distributions will be supported in future releases

  • A Git Repository: A Git repository for storing GitOps configurations (GitHub, GitLab, Bitbucket, or any Git provider)

Bootstrap Process

The Service Foundry Bootstrap tool automates the entire installation process:

  1. Initial Setup: Installs ArgoCD as the GitOps engine on your Kubernetes cluster

  2. Core Component Deployment: Uses ArgoCD to deploy all core components from your GitOps repository

  3. GitOps Configuration: Creates the necessary repository structure and manifests for managing your platform

  4. Console Activation: Deploys the Service Foundry Console UI for managing applications and configurations

All core components are version-controlled in the GitOps repository, allowing you to:

  • Track all changes through Git commit history

  • Review and approve infrastructure changes via pull requests

  • Update components using standard GitOps workflows

  • Roll back to previous configurations when needed

  • Maintain consistency across multiple environments

Service Foundry Profiles

Service Foundry Profiles are pre-configured, production-ready stacks of integrated open-source tools designed for specific use cases. Each profile includes tested configurations, best practices, and documentation to accelerate your platform adoption.

Available Profiles

  • GitOps Profile: Pre-configured ArgoCD and Argo Rollouts with:

    • Multi-environment deployment pipelines (dev, staging, production)

    • Automated promotion workflows

    • Rollback and progressive delivery strategies

  • Application Catalog Profile: Curated collection of popular open-source applications including:

    • Databases (PostgreSQL, MySQL, MongoDB, Redis)

    • Message queues (RabbitMQ, Kafka)

    • Caching solutions (Redis, Memcached)

    • Development tools (GitLab, Jenkins)

  • Observability Profile: Complete observability stack with pre-configured integrations between:

    • OpenTelemetry, Prometheus, Grafana, Jaeger

    • OpenSearch, Fluent Bit, Loki, Tempo

    • Pre-built dashboards and alerting rules

  • Security and SSO Profile: Enterprise security stack featuring:

    • Keycloak identity management with user federation

    • OAuth2 Proxy for application protection

    • Traefik Gateway API for secure ingress

    • Sealed Secrets for encrypted credentials

  • Data Pipelines Profile: Data engineering and processing stack including:

    • Apache Airflow for workflow orchestration

    • Apache Spark for distributed data processing

    • Apache Kafka for event streaming

    • Neo4j graph database

    • PostgreSQL for data warehousing

    • Redis for data caching

Each profile is designed to work standalone or in combination, allowing you to build a complete platform tailored to your team’s needs.

Who is Service Foundry For?

Service Foundry is designed for teams and organizations looking to adopt GitOps practices on Kubernetes without the overhead of assembling and integrating multiple tools.

Ideal Use Cases

  • Platform Engineering Teams: Build internal developer platforms with standardized workflows and self-service capabilities

  • Startups and Scale-ups: Establish production-grade infrastructure quickly without dedicating extensive resources to platform development

  • Educational Institutions: Provide students and researchers with a complete, enterprise-grade technology stack for learning modern DevOps practices, including GitOps, observability, security, and data engineering

  • Innovation and PoC Teams: Rapidly prototype and validate new technologies on Kubernetes with pre-configured, production-ready components

  • Enterprise Development Teams: Standardize application deployment pipelines and infrastructure management across multiple teams and projects

  • DevOps Practitioners: Implement GitOps best practices with proven patterns and tools without starting from scratch

Best Practices

To maximize the value of Service Foundry, we recommend the following practices:

  • Version Control Everything: Store all configuration for platform components and enterprise applications in Git repositories with proper version control

  • Implement Automated Promotion Pipelines: Establish progressive delivery workflows across environments (PR → Development → Staging → Production) with automated testing gates

  • Enforce Authentication: Use SSO (Single Sign-On) for all platform and application access to maintain centralized identity management

  • Secure Sensitive Data: Store all secrets using Sealed Secrets or external secret managers, never committing plain-text credentials to Git

  • Use Gateway API: Leverage Traefik’s Gateway API implementation for standardized, secure external access to services

  • Enable Observability: Deploy the observability stack in all non-development environments to maintain visibility into system health and performance

  • Regular Updates: Keep platform components updated through GitOps workflows, testing updates in lower environments before production

  • Document Custom Configurations: Maintain documentation for any customizations or extensions to Service Foundry profiles in your GitOps repository

Step 2: Download the Bootstrap Package

The Service Foundry Bootstrap Package contains everything you need to deploy the platform on AWS EKS.

  1. From the same community portal (https://community.servicefoundry.org/), navigate to Bootstrap Scripts

  2. Download the bootstrap package as a ZIP file

  3. Extract the contents to your working directory

Figure 2. Download the Bootstrap Scripts

What’s Included in the Bootstrap Package

The downloaded ZIP file contains the following components:

  • License File: Your Community Edition license (if included in the package, otherwise use the separately downloaded .lic file)

  • Bootstrap Scripts: Automated installation scripts for deploying Service Foundry core components

  • Policy Files: AWS IAM policies and Kubernetes RBAC configurations

  • EKS Setup Scripts: AWS CLI scripts to provision and configure an Amazon EKS cluster

  • Configuration Templates: Customizable YAML files for platform configuration

  • Documentation: Quick start guide and installation instructions

Figure 3. The Bootstrap Package Contents

Step 3: Prepare Your Environment

Before running the bootstrap scripts, ensure you have the following prerequisites:

Required Tools

  • AWS CLI: Version 2.x or higher, configured with appropriate credentials

  • kubectl: Kubernetes command-line tool (v1.33+)

  • eksctl: (Optional but recommended) AWS EKS cluster management tool

  • Helm: Kubernetes package manager (v3.10+)

  • Git: For cloning your GitOps repository

Execution Environment Options

You can run the bootstrap scripts from any of the following environments:

  • AWS CloudShell: Recommended for quick setup—pre-configured with AWS CLI and kubectl

  • Local Machine: Your laptop or workstation with AWS CLI configured

  • EC2 Instance: A dedicated AWS instance with appropriate IAM roles

  • CI/CD Pipeline: Automated deployment through your existing pipeline

AWS CloudShell is the recommended option for first-time users as it requires no local setup and comes pre-authenticated with your AWS credentials.

Next Steps

After downloading the license and bootstrap package:

  1. Review the included README.md for detailed installation instructions

  2. Ensure your AWS account has the necessary permissions (see IAM policy files included in the package)

  3. Prepare your Git repository for GitOps configuration storage

  4. Follow the bootstrap installation guide to deploy Service Foundry
