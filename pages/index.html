Key Features

Automated Provisioning and GitOps Management

Service Foundry automatically generates all the required Kubernetes manifest files through its built-in generators. GitOps workflows are enabled by default using ArgoCD, ensuring your Kubernetes cluster always stays in sync with your Git repository. All ArgoCD applications and related Kubernetes resources are stored in a single Git repository, making it simple to manage, version, and track changes across your environment.

Kubernetes Native Service Foundry Builder

No installation needed. Service Foundry Builder is a Kubernetes-native application that runs on your cluster. It uses the same configuration as your applications, so you can deploy and manage everything in one place.

Automated Deployment of App Observability Platform

Provision a full observability platform on Kubernetes with a single configuration file. This includes OpenTelemetry, Jaeger, Prometheus, Grafana, and more.

Single Configuration, Full Stack Deployment

Deploy a complete platform stack with just one configuration file. Service Foundry automates the setup of all components, including SSO, observability, and big data tools.

Modular Architecture

Use Service Foundry as a whole or select individual components to fit your needs. Each module integrates smoothly with your existing Kubernetes setup.

Simplified Deployment Process

Reduce operational complexity. Service Foundry automates installation and configuration, enabling seamless scaling and environment management.

Unified Authentication with SSO

Enable Single Sign-On for all your applications through one authentication provider. Enhance user experience and strengthen security â€” no more credential fatigue.

Cloud Provider Flexibility

Service Foundry is Kubernetes provider agnostic. Whether you use AWS (EKS), Azure (AKS), or Google Cloud (GKE), deployment works consistently across environments.
